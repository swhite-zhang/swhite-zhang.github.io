插入排序

最简单的排序算法。将一个记录插入到已排好序的有序表中，从而的导演给新的、记录新增1的有序表。

复杂度

时间复杂度为O(n²)。最好的情况为比较n-1次，移动0次。最坏的情况为比较n²/4次，移动n²/2次。

算法

步骤

  1. 从序列中得到第一个数
  2. 将这个数插入到新序列中合适的位置，并将其从原序列中去除
  3. 从序列中选取下一个数
  4. 重复步骤2，3直到排序完成

二路插入排序

插入排序的改进版。为了减少插入过程中的移动，二路插入排序算法使用了一个循环数组。将基数放在第一个位置，比基数小的将会被插入到基数前面，否则被放到基数的后面，最后将这个序列的最小的数移到第一位。

算法

步骤

  1. 从序列中选取一个数
  2. 将这个数与基数比较，如果小就放在基数左边的相应位置，否则放在右边的相应位置
  3. 从序列中选取下一个数
  4. 重复2，3步骤直到排序完成

希尔排序

又称缩小增量排序，是插入算法的改进版。它将待排记录序列分割为若干子序列分别进行插入排序。需要注意，希尔排序是不稳定的。

复杂度

  • 时间复杂度：O(n^1.25~1.6)
  • 空间复杂度：O(1)

算法

步骤

  1. 选择一个间隔d
  2. 从序列抽取第一个数
  3. 将这个数插入到间隔尾d的一串数中合适的位置
  4. 从序列中向后抽取第d个数
  5. 重复步骤3，4直到抽取最后一个数
  6. 将d除以2并下取整
  7. 重复步骤1，2，3，4，5，6直到d为0

冒泡排序

一个简单但是复杂度高的算法。

复杂度

时间复杂度为O(n²)。

算法

步骤

  1. 选择第一个数
  2. 将当前的数与后一个数比较，如果大于后一个数，则将两个数交换位置
  3. 选取后一个数
  4. 重复步骤2，3直到排序完成

快速排序

通过一趟排序将待排序列分割成独立的两个部分，其中一部分的数均比另一部分小，然后再对这两个部分进行快排，从而达到排序的目的。

复杂度

时间复杂度为O(nlog2n)，最坏为O(n²)。

算法

步骤

  1. 任意选取一个记录作为枢轴
  2. 将记录中小于枢轴的值列在左边，将大于枢轴的值列在右边
  3. 对得到的两个部分分别进行快速排序直到每个部分都只有一个数

算法

选择排序

遍历序列，选择最大的和最小的分别列于队首和队尾。

复杂度

时间复杂度为O(n²)

算法

步骤

  1. 遍历序列，选择序列中最大的数和最小的数
  2. 将最小的数移到队首，最大的数移到队尾，并将这个两数移出序列
  3. 重复步骤1，2直到排序完成

堆排序

利用堆这种数据结构所设计的一种排序算法。堆是一个近似完全二叉树的结构，并同时满足堆的性质：即子节点的键值或索引总是小于（或者大于）它的父节点。

复杂度

时间复杂度：O(nlog2n)

算法

步骤

  1. 用筛选的到一个大顶堆
  2. 将最大的记录与最后一个记录交换
  3. 用筛选使剩余的序列又成为一个大顶堆
  4. 重复步骤2，3直到只剩最后一个记录
